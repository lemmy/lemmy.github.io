uebersichtlichkeit von todo tags in eclipse

die standard templates von eclipse generieren bei neu erzeugte methoden automatisch einen todo tag. leider aendert man aus zeitmangel seltents den default hinweis und so schwillt die todo liste schnell auf x identische eintraege an. ausserdem veraendern todo tags nicht das laufzeitverhalten des codes und so bemerkt man ggf. nicht, dass eine methode noch gar nicht implementiert ist. daher scheint es mir sinnvoller, anstatt einen todo tag ein "throws new NotImplementedException()" im code template vorzusehen. NotImplementedException erbt dann von RuntimeException. nun wirkt sich die unvollstaendigkeit des codes auch aufs laufzeitverhalten aus und man bemerkt, dass noch arbeit aussteht.
